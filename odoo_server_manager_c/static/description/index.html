Start, Stop, Restart | Odoo Server Manager

The Odoo Restart module allows the administrator to start, stop, and restart the odoo server using command-line functions. The administrator can manage multiple servers and can specify details such as the username, client name, server URL or IP, port, and password. This is useful when the client encounters problems and needs to update various server actions.

The Odoo Restart module allows the administrator to list the credentials for the server of different clients. Additionally, as the server action commands (start, stop, restart) vary according to the operating system, the administrator can configure the commands for starting/stopping/restarting accordingly. When the server action button is clicked, the mentioned command will be successfully executed implementing the action. The administrator can also track the server actions performed for multiple client servers through the history management function. Additional information can be mentioned in the instructions tab if needed.

Requirements to be met prior to installation

pexpect: sudo pip3 install pexpect

Configuration

Only users with access rights to 'Odoo Restart' can manage server actions. Go to Settings > User > Other > Odoo Restart and add server details.

As you can see in the image, the administrator can add multiple client servers with various details such as username, password, server URL, port, and client name. Separate user groups are created so that server action can be easily managed, and the administrator can also provide access to a particular user to manage server action. Go to Odoo Restart -> Create -> Add details.

After listing the details of the client server, the administrator, in case of any difficulty with the client server, can perform various actions on the server, such as START, STOP, and RESTART. Additionally, the administrator can provide commands for server actions based on the operating system. When the server action is clicked, the particular command will be executed in the backend and the problem with the client server will be easily resolved without the need to connect to the Odoo terminal.

All actions taken can be managed through the history. The administrator can view the name of the client server, created by which user, and the server action taken on that server.
